Vì sao sửa thẳng ZoneDNS là an toàn nhất
Domain đang dùng DNS của ZoneDNS, và bạn đã có quyền sửa record trực tiếp. Để trỏ web sang Vercel, chỉ cần sửa đúng 1 record A — không chuyển nền tảng, không đổi nameserver, không chép lại record email.

3
Đưa WordPress cũ về old.dhtcdanang.com
~30-45 phút · Sửa data WP · Làm TRƯỚC cutover
Todo ›
Trước khi apex trỏ sang Vercel, ta phải "dời" web cũ sang địa chỉ old.dhtcdanang.com và làm cho nó chạy độc lập. Nếu bỏ bước này, sau cutover WordPress sẽ không còn truy cập được.
cPanel bản mới đã bỏ icon Subdomains riêng — tạo subdomain ở mục Domains:
  • cPanel → Domains → nút Create A New Domain
  • Domain: gõ đầy đủ old.dhtcdanang.com (cPanel tự nhận đây là subdomain của domain chính)
  • TICK ô "Share document root (/home/nhdhtq8m/public_html) with dhtcdanang.com" → để old. serve ĐÚNG bộ WordPress hiện tại. Nếu KHÔNG tick, cPanel tạo thư mục old.dhtcdanang.com trống → mở ra trắng trơn.
  • Bấm Submit
ZoneDNS KHÔNG tạo được subdomain này
ZoneDNS chỉ trỏ tên old.dhtcdanang.com về IP (bước 3.2). Còn việc máy chủ cPanel "nhận" hostname đó và serve đúng WordPress thì phải khai báo trong cPanel (bước này). DNS và web server là 2 lớp khác nhau — thiếu lớp cPanel thì old. chỉ trỏ về IP nhưng không hiện site.
⚠ "Share document root" là vĩnh viễn — đúng ý ta
cPanel ghi setting này không đổi được sau. Đúng mục tiêu: old. dùng chung file WordPress với apex (chỉ 1 bản WP duy nhất). Hệ quả: khi đổi URL ở bước 3.6, apex cũ cũng redirect sang old. — chấp nhận được vì sắp cutover apex sang Vercel.
zonedns.vn → Quản lý bản ghi → thêm A record:
HostTypeValueTTL
oldA103.28.36.220300

⛔ KHÔNG dùng "Zone Editor" trong cPanel
cPanel có mục Zone Editor trông y hệt chỗ sửa DNS — nhưng domain đang chạy nameserver ns1-4.zonedns.vn, nên DNS thật nằm ở ZoneDNS. Sửa trong cPanel Zone Editor sẽ không có tác dụng (đó là lý do zone trong cPanel chỉ thấy 1 record, còn ZoneDNS có 5). Luôn thêm/sửa ở zonedns.vn. Và record này phải là Type A — không phải MX.
Đợi ~5-10 phút rồi kiểm tra:
dig old.dhtcdanang.com +short # phải ra 103.28.36.220
Mở http://old.dhtcdanang.com → phải thấy WordPress (có thể chưa có SSL, dùng http tạm).
cPanel → SSL/TLS Status → tick old.dhtcdanang.com → Run AutoSSL (Let's Encrypt miễn phí). Đợi vài phút → https://old.dhtcdanang.com chạy.
Bước 3.6 cần vào wp-admin. Nếu chưa biết / quên mật khẩu WP → KHÔNG đoán mò, dùng 1 trong 3 cách (ưu tiên trên xuống):
  1. cPanel → WordPress Management (có sẵn ở menu trái) → chọn site dhtcdanang.com → nút Log In → vào thẳng wp-admin không cần mật khẩu. Tại đây cũng xem/đổi được admin user.
  2. Quên mật khẩu qua email: trang wp-login.php → bấm "Lost your password?" → nhập email admin → nhận link reset (email @dhtcdanang.com đang sống nên nhận được).
  3. Reset qua phpMyAdmin: cPanel → phpMyAdmin → DB WordPress → bảng wp_users → Edit user admin → cột user_pass chọn function MD5 + nhập mật khẩu mới → Go.
Đăng nhập ở https://dhtcdanang.com/wp-admin (apex vẫn trỏ cPanel + đã có SSL). Chưa cần old. để login.
Cả 3 cách trên đều không vào được? → Bỏ qua 3.5 + 3.6, dùng bước 3.6-ALT bên dưới: đổi URL bằng wp-config.php qua File Manager — không cần wp-admin.
⚠ Backup DB trước (đã có ở Phase 1) — bước này sửa database
Chỉ làm SAU khi old. đã resolve (3.3). Nếu đổi URL khi old. chưa sống → mất luôn truy cập wp-admin.
Cách an toàn — plugin:
  1. WP admin (dhtcdanang.com/wp-admin) → Plugins → cài "Better Search Replace"
  2. Tools → Better Search Replace
  3. Search: https://dhtcdanang.com → Replace: https://old.dhtcdanang.com
  4. Chọn tất cả bảng → tick "Run as dry run" trước để xem số thay đổi
  5. Bỏ dry run → Run Search/Replace
Hoặc nhanh hơn (nếu chỉ cần): WP admin → Settings → General → đổi WordPress Address (URL) + Site Address (URL) → https://old.dhtcdanang.com → Save.
Khi nào dùng bước này
Cả 3 cách ở bước 3.5 đều không vào được wp-admin (cPanel không có nút Log In, email reset không tới, phpMyAdmin reset xong vẫn không login được vì 2FA hoặc lỗi khác). Bước này dùng chỉ cPanel File Manager — không cần wp-admin, không sửa database.
Vì sao chạy được: WordPress đọc 2 hằng số WP_HOME + WP_SITEURL trong wp-config.php ưu tiên hơn giá trị trong DB (wp_options.siteurl/home). Định nghĩa 2 dòng này → URL chính của site đổi ngay, không cần wp-admin, không sửa wp_options.
  1. Backup file gốc — File Manager → public_html → wp-config.php → chuột phải → Copy → đặt tên wp-config.bak.php. Cần để rollback nhanh.
  2. File Manager → public_html → wp-config.php → chuột phải → Edit (hộp thoại encoding → giữ mặc định utf-8 → Edit).
  3. Tìm dòng comment /* That's all, stop editing! Happy publishing. */ (gần cuối file). Chèn ngay PHÍA TRÊN dòng đó:
define( 'WP_HOME', 'https://old.dhtcdanang.com' ); define( 'WP_SITEURL', 'https://old.dhtcdanang.com' ); # 2 dòng — không có https quá hạn, không có dấu / cuối
  1. Save Changes ở góc trên phải editor.
  2. Mở https://old.dhtcdanang.com trong tab ẩn danh → trang chủ hiển thị, URL trên thanh địa chỉ luôn là old.dhtcdanang.com (không bị 301 về dhtcdanang.com).
  3. Thử https://old.dhtcdanang.com/wp-login.php → trang login hiện ra (chưa cần đăng nhập, chỉ cần URL không 404 / không redirect sai).
Giới hạn — đọc trước khi chạy bước 3.7
Hằng số chỉ ép siteurl + home. URL kiểu https://dhtcdanang.com/wp-content/uploads/... hardcode trong:
  • nội dung bài post (cột post_content trong wp_posts)
  • widget / theme options (serialized PHP trong wp_options)
  • menu items, customizer settings
vẫn trỏ về dhtcdanang.com — sau cutover các URL này sẽ rớt sang Vercel (404). Nếu chỉ là archive đọc hiếm khi click vào ảnh/menu → bỏ qua, chấp nhận. Nếu cần old.* trông sạch → làm thêm bước phụ bên dưới.
Tuỳ chọn — thay thế URL trong DB không cần wp-admin (serialized-safe)
  1. Tải về tool open-source Search-Replace-DB của interconnect/it: https://interconnectit.com/products/search-and-replace-for-wordpress-databases/ → giải nén lấy thư mục Search-Replace-DB-master.
  2. Đổi tên thư mục thành chuỗi ngẫu nhiên (vd srdb-9k2x) để URL khó đoán.
  3. File Manager → public_html → Upload → tải toàn bộ thư mục lên.
  4. Trình duyệt mở https://old.dhtcdanang.com/srdb-9k2x/ → tool tự đọc wp-config.php để connect DB (không nhập tay password).
  5. search for: https://dhtcdanang.com · replace with: https://old.dhtcdanang.com
  6. Bấm Dry run trước → xem số dòng sẽ đổi → nếu OK, bấm Live run.
  7. BẮT BUỘC sau khi xong: bấm Delete me trong tool, hoặc File Manager → xoá thư mục srdb-9k2x. Tool còn nằm trên server = lỗ hổng nghiêm trọng.
Rollback (nếu cần huỷ ALT)
File Manager → xoá 2 dòng define(...) vừa thêm (hoặc rename wp-config.bak.php → wp-config.php, ghi đè). DB chưa bị đụng → WP về lại URL cũ ngay lập tức. Nếu đã chạy SRDB live run → restore DB từ backup Phase 1.4.
Mở https://old.dhtcdanang.com → trang chủ, 1 bài post, ảnh load đúng. Login old.dhtcdanang.com/wp-admin được.
Lúc này apex dhtcdanang.com (vẫn trỏ cPanel) sẽ redirect sang old.* — đúng như mong đợi. Sau Phase 4, apex sẽ thành Vercel.
