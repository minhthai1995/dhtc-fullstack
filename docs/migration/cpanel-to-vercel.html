Backup full cPanel
~30-60 phút · BẮT BUỘC làm đầu tiên · Zero risk
Todo ›
Snapshot toàn bộ data trước khi đụng bất cứ thứ gì. Nếu sau này hosting cũ ngừng hoạt động hoặc cutover hỏng, đây là bản phục hồi duy nhất. Đừng skip.
Mở https://dhtcdanang.com:2083 hoặc https://103.28.36.220:2083. Login với username nhdhtq8m.
  1. Tìm icon "Backup" hoặc "Backup Wizard" trong cPanel
  2. Chọn Full Backup
  3. Destination: Home Directory
  4. Email khi xong: điền email cá nhân của bạn
  5. Submit → đợi 5-30 phút (cPanel gửi mail "Backup complete")
File Manager → root directory → tìm file backup-MM.DD.YYYY_HH-MM-SS_nhdhtq8m.tar.gz → click Download. File này chứa toàn bộ public_html, mọi MySQL DB, email, DNS zone, cron jobs.
⚠ Lưu file ở 2 nơi
Máy local + 1 ổ đĩa cloud (Google Drive / iCloud / OneDrive). File này không thay thế được nếu mất.
  1. cPanel → phpMyAdmin
  2. Chọn DB trong sidebar trái (thường tên nhdhtq8m_wp... hoặc nhdhtq8m_db...)
  3. Tab Export → Method: Quick → Format: SQL
  4. Click Go → tải file .sql về
Nếu có nhiều DB → export từng cái.
File Manager → vào public_html → click Select All → click Compress → chọn Zip → đợi xong → Download file .zip về.
  • cPanel → Email Accounts → screenshot list (tên + quota)
  • cPanel → Forwarders → screenshot
  • cPanel → Email Routing → screenshot
Sau migration nếu cần tạo lại email ở dịch vụ khác (Google Workspace / Zoho) thì có reference.
  1. cPanel → Zone Editor
  2. Click domain dhtcdanang.com → Manage
  3. Screenshot TOÀN BỘ records (A, AAAA, MX, TXT, CNAME, SRV)
  4. Nếu có nút Export Zone File → tải file .txt về
⚠ Đặc biệt chú ý ghi lại:
  • Tất cả MX records (priority + hostname)
  • Mọi TXT record chứa v=spf1, _domainkey, _dmarc
  • TXT google-site-verification hoặc facebook-domain-verification nếu có
  • A record mail.dhtcdanang.com (nếu có)
Phase 6 sẽ cần check lại để KHÔNG xoá nhầm.
✓ Sau Phase 1 bạn có:
File .tar.gz backup full + file .sql DB + zip public_html + screenshot DNS + email list. Đây là snapshot rollback hoàn chỉnh.

Giảm TTL ở Squarespace
~10 phút thao tác + đợi 1h · Zero risk · Prep cho rollback nhanh
Todo ›
TTL hiện tại (thường 1h-24h) là thời gian DNS cache toàn cầu nhớ giá trị cũ. Giảm TTL TRƯỚC khi cutover để nếu Phase 6 gãy thì rollback chỉ mất 5 phút thay vì 24h.
Mở https://account.squarespace.com/domains → login.
Chọn đúng domain
URL bạn có là .../fdvvietnam.com/dns/dns-settings — đó là URL của domain khác trong cùng account. Trong dashboard, chọn dhtcdanang.com từ list → DNS Settings.
Trong khu Custom Records hoặc DNS Records:
  • Tìm record có Type = A, Host = @ (hoặc trống)
  • Value hiện tại đang là 103.28.36.220
  • Click Edit
  • Đổi TTL xuống 5 min (300 giây)
  • GIỮ value 103.28.36.220 — chỉ đổi TTL
  • Save
  • Record có Type = CNAME, Host = www
  • Edit → TTL xuống 5 min
  • GIỮ value cũ — chỉ đổi TTL
  • Save
DNS cache cũ ở các ISP toàn cầu cần thời gian expire xuống TTL mới. Nếu cutover ngay sẽ không có lợi ích.
Trong lúc đợi có thể làm Phase 3 (subdomain) — không ảnh hưởng đến apex.
3
Setup old.dhtcdanang.com
~30 phút · Low risk · Chỉ ADD record mới, không sửa gì
Todo ›
Tạo "lối thứ 2" vào WordPress qua subdomain trước khi cutover apex. Sau khi apex đã chuyển sang Vercel, WP vẫn còn truy cập được ở old.*.
  1. Login cPanel → tìm icon Subdomains
  2. Form:
    • Subdomain: old
    • Domain: dhtcdanang.com
    • Document Root: public_html (cùng folder, KHÔNG tạo folder mới)
  3. Click Create
Quan trọng: Document Root = public_html
Để old.* serve cùng WordPress files với dhtcdanang.com. Không tạo folder mới (vd public_html/old).
Vẫn ở Squarespace DNS settings của dhtcdanang.com → Add Record:
  • Type: A
  • Host: old
  • Value: 103.28.36.220
  • TTL: 1 hour
Save.
Mở Terminal (Mac/Linux) hoặc whatsmydns.net nếu Windows:
dig old.dhtcdanang.com +short # expect: 103.28.36.220
Nội dung WP phải hiện ra. Có thể ảnh/CSS vỡ vì URL hard-code https://dhtcdanang.com/.... Bình thường — Phase 4 sẽ fix.
Nếu trắng trang hoặc redirect về dhtcdanang.com: vào cPanel → File Manager → mở public_html/.htaccess coi có rule canonical redirect không. Cũng có thể WP option siteurl/home hard-code domain chính → Phase 4 sẽ sửa.
4
WordPress URL search-replace
~30-45 phút · Medium risk · Sửa WP DB (backup Phase 1 là safety net)
Todo ›
WP lưu URL absolute trong DB (posts, options, theme settings, ACF, widgets...). Nếu không replace, sau Phase 6 các URL https://dhtcdanang.com/wp-content/... sẽ trỏ sang Vercel → 404 hết.
⚠ BẮT BUỘC làm Phase 1 (backup DB) trước
Search-replace lỡ tay là hỏng toàn bộ WP. Nếu chưa export .sql từ phpMyAdmin → quay lại Phase 1.4.
https://dhtcdanang.com/wp-admin (làm trước khi switch DNS — sau Phase 6 thì login qua https://old.dhtcdanang.com/wp-admin).
Plugins → Add New → search "Better Search Replace" (tác giả WP Engine) → Install Now → Activate.
  1. Tools → Better Search Replace
  2. Search for: https://dhtcdanang.com
  3. Replace with: https://old.dhtcdanang.com
  4. Select tables: Cmd/Ctrl + click chọn TẤT CẢ tables (kéo từ đầu list đến cuối)
  5. ✅ Run as dry run? (check để xem trước, KHÔNG ghi DB)
  6. Click Run Search/Replace
  7. Coi kết quả: bao nhiêu cells phải đổi — số > 0 là đúng (thường 20-200 cells)
Bỏ ✅ Run as dry run? → Run Search/Replace → đợi vài giây → báo "X tables were searched, Y cells were changed".
  • Search: http://dhtcdanang.com
  • Replace: http://old.dhtcdanang.com
  • Dry run → Run thật
  • Search: //dhtcdanang.com
  • Replace: //old.dhtcdanang.com
  • Dry run → Run thật
Cẩn thận: pattern này có thể match cả URL bên thứ 3 (vd //ajax.googleapis.com). Coi dry run kỹ — nếu thấy host lạ thì skip.
WP admin → Settings → General:
  • WordPress Address (URL): http://old.dhtcdanang.com
  • Site Address (URL): http://old.dhtcdanang.com
  • Save Changes
Sau khi save, WP tự logout
Đăng nhập lại qua http://old.dhtcdanang.com/wp-admin (không phải dhtcdanang.com nữa).
Click các trang khác nhau (homepage, blog post, contact...) → ảnh/CSS phải render đúng, link nội bộ phải point về old.*. Nếu còn URL dhtcdanang.com trong source view → lặp lại Phase 4.3 cho pattern còn sót (vd có space khác, encoding khác).
5
Add domain vào Vercel
~10 phút · Zero risk · Vercel "ngồi chờ" cho đến Phase 6
Todo ›
Mở https://vercel.com/dashboard → click vào project DHTC (project đang serve preview build).
  1. Tab Settings → menu trái Domains
  2. Input: dhtcdanang.com → click Add
  3. Vercel hỏi cách dùng www → chọn "Redirect www.dhtcdanang.com to dhtcdanang.com" (apex là primary)
Vercel hiện banner "Invalid Configuration" với 2 DNS values cần set:
  • A record apex: 76.76.21.21
  • CNAME www: cname.vercel-dns.com
"Invalid Configuration" là bình thường — chưa đụng DNS Squarespace. Vercel sẽ tự verify khi DNS propagate sau Phase 6.
Vì DHTC là React SPA, mọi route /admin, /privacy, /terms phải fallback về index.html. File vercel.json trong repo nên có rewrite này (đã có sẵn từ deploy lần đầu — kiểm tra cho chắc):
{ "rewrites": [ { "source": "/(.*)", "destination": "/index.html" } ] }
6
DNS Cutover (moment switch thật)
~10 phút thao tác + 10 phút verify · HIGH RISK · Đêm khuya / sáng sớm
Todo ›
Đây là phase quan trọng nhất. Sau khi save DNS, user truy cập dhtcdanang.com sẽ thấy React app thay vì WordPress. Có downtime ngắn trong khi DNS propagate (5-30 phút tuỳ ISP).
Pre-check trước khi click
  • Phase 1 backup xong (file .tar.gz về máy)
  • Phase 2 TTL = 5 min + đã đợi 1h
  • Phase 3 old.dhtcdanang.com hoạt động
  • Phase 4 WP đã search-replace, mở được tại old.*
  • Phase 5 domain đã add trong Vercel
  • Có sẵn 30 phút thời gian liền mạch để verify
Đêm khuya (sau 23:00) hoặc sáng sớm (trước 7:00) — fanpage check-in ít. Tránh giờ cao điểm 18:00-22:00.
  1. Vào DNS Settings của dhtcdanang.com
  2. Tìm A record host = @ (value đang là 103.28.36.220)
  3. Edit → Value: 76.76.21.21
  4. TTL: vẫn 5 min
  5. Save
  1. Tìm CNAME host = www
  2. Edit → Value: cname.vercel-dns.com
  3. TTL: 5 min
  4. Save
Squarespace có thể yêu cầu trailing dot cname.vercel-dns.com. Một số UI tự add. Save thử cả 2 format nếu báo lỗi.
  • ❌ MX records — đụng = mất email
  • ❌ TXT chứa v=spf1 — đụng = email vào spam
  • ❌ TXT _domainkey (DKIM) — đụng = email bị reject
  • ❌ TXT _dmarc — đụng = email auth fail
  • ❌ TXT google-site-verification — đụng = mất Google Search Console
  • ❌ A record mail (nếu có)
  • ❌ A record old đã add ở Phase 3
# Apex phải về Vercel dig dhtcdanang.com +short # expect: 76.76.21.21 # WWW phải về Vercel CNAME dig www.dhtcdanang.com +short # expect: cname.vercel-dns.com. # 76.76.21.21 # MX KHÔNG đổi — email server cũ dig MX dhtcdanang.com +short # expect: số priority + hostname như trước Phase 6 # Old vẫn về cPanel dig old.dhtcdanang.com +short # expect: 103.28.36.220
Nếu DNS chưa propagate ở khu vực bạn → check tại whatsmydns.net (xem propagation toàn cầu).
  • https://dhtcdanang.com → React app mới (Vercel)
  • https://www.dhtcdanang.com → redirect về apex
  • https://old.dhtcdanang.com → WordPress cũ
Nếu apex vẫn ra WP cũ → DNS chưa propagate ở máy bạn. Đợi thêm 5-10 phút. Có thể flush DNS:
# macOS sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
Gửi 1 email test từ Gmail/Outlook cá nhân tới info@dhtcdanang.com hoặc bất kỳ mailbox đang active. Login webmail tại https://dhtcdanang.com:2096 (hoặc 103.28.36.220:2096) → check inbox.
Nếu email không nhận được sau 5 phút → kiểm tra MX records trong Squarespace có giữ nguyên không. So với screenshot Phase 1.7.
7
SSL + finalize
~30 phút · Low risk · Cert auto-issue
Todo ›
Vercel tự xin Let's Encrypt sau khi DNS propagate. Mở Settings → Domains → trạng thái phải chuyển từ "Invalid Configuration" → "Valid Configuration" + ✅ HTTPS active.
Đợi 5-30 phút. Sau đó browse https://dhtcdanang.com không có warning cert.
  1. cPanel → SSL/TLS Status
  2. Tìm row old.dhtcdanang.com (có thể có ❌ status)
  3. Click Run AutoSSL trên row đó (hoặc Run AutoSSL for All Domains ở trên)
  4. Đợi 5-15 phút → status đổi ✅
AutoSSL cần Let's Encrypt verify được http://old.dhtcdanang.com/.well-known/acme-challenge/... — phải DNS đã propagate xong (Phase 3).
  • ✅ https://dhtcdanang.com — React app, cert hợp lệ
  • ✅ https://www.dhtcdanang.com — redirect apex, cert hợp lệ
  • ✅ https://old.dhtcdanang.com — WP, cert hợp lệ
Squarespace DNS → A @ + CNAME www: TTL 5 min → 1 hour. Lý do: TTL 5 min làm tăng load lên DNS server, không cần thiết khi đã ổn định.
⟲ Rollback — nếu Phase 6 gãy

Vì Phase 2 đã hạ TTL xuống 5 phút từ trước, rollback chỉ mất < 5 phút.

  1. Vào Squarespace DNS Settings của dhtcdanang.com
  2. A record @ → Value: 76.76.21.21 → đổi LẠI thành 103.28.36.220 → Save
  3. CNAME www → Value: cname.vercel-dns.com → đổi LẠI về value cũ (xem screenshot Phase 1.7) → Save
  4. Đợi 5 phút → dig dhtcdanang.com +short ra 103.28.36.220 = về như cũ
  5. WordPress mở qua https://dhtcdanang.com hoạt động bình thường

Lưu ý: nếu đã làm Phase 4 (WP search-replace), URL trong WP đang là old.*. Sau rollback có thể cần search-replace ngược lại old.dhtcdanang.com → dhtcdanang.com, hoặc restore DB từ file .sql backup Phase 1.4.

Post-migration checklist (1-2 ngày sau)
Add property mới cho https://dhtcdanang.com (Vercel) — verify qua HTML tag hoặc TXT record. Submit sitemap https://dhtcdanang.com/sitemap.xml (nếu React app generate).
Chọn 1 trong 2:
  • Noindex: WP admin → Settings → Reading → ✅ "Discourage search engines from indexing this site". Bài cũ rớt index sau 1-2 tuần.
  • 301 redirect: edit public_html/.htaccess add rule redirect URL cũ → URL React app tương đương. Giữ SEO juice.
  • Facebook Fanpage — About section
  • Google Business Profile
  • Name card / brochure đã in
  • Email signature
  • QR code (nếu có in trên menu, biển hiệu)
cPanel host hiện vẫn cần để:
  • Serve old.dhtcdanang.com (WP cũ)
  • Nhận email @dhtcdanang.com
Trừ khi migrate email sang Google Workspace / Zoho + xoá WP cũ → vẫn cần duy trì cPanel. Coi ngày hết hạn hosting, plan trước.
